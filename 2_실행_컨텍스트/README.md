 > **은지**: 호이스팅과 스코프가 중요했음 식별자를 최상단으로 올려진 것 처럼
 > 어디서 할당되는 지에 따라 코드 내용이 달라지니까. 
 > VE와 RE는 거의 이해를 못했음 어려웠음
 
 > **유진**: 함수 호이스팅 개념이 중요하다는 생각을 하였음.
 > 함수 선언문과 함수 표현식에서의 중요성의 이유 예시
 > 전역 변수보단 지역 변수 사용의 중요성을 꺠달았다.
 > 어려웠던 점: VE, RE, This바인딩 부분이 어려웠음

 > **혜성**: 중요하다고 생각한점: 호이스팅 개념 부분(면접 진행 시)
 > let과 const를 쓰기 떄문에 개념을 알면 좋지 실질적으로 개발에 쓰일 것 같진 않다.
 > 정보 블로그: [https://medium.com/korbit-engineering/let%EA%B3%BC-const%EB%8A%94-%ED%98%B8%EC%9D%B4%EC%8A%A4%ED%8C%85-%EB%90%A0%EA%B9%8C-72fcf2fac365]
 > 수집을 해서 호이스팅이 일어난다. key point (환경 부분)
 > 함수 선언문과 표현식. arrow function을 요즘엔 더 쓴다.
 > react를 쓴다거나 다른 걸 쓸 때 함수 선언문과 표현식의 차이가 크지는 않다.
 > 파일들을 한데 뭉쳐야되는데 번들링, 모듈 이런 개념 때문에 차이가 크게 중요하진 않지만
 > 원리를 알고 있으면 도움이 될 것 같다 !
 > 몰랐던 점: 실행 컨텍스트의 나눠지는 부분들.
 > This는 추후 3장에서 

 > **효중**: 호이스팅, 실행 컨텍스트의 구성이 중요했다.
 > 표현식과 선언문이 변수 부분만 호이스팅 된다는 것. 이 부분을 새로 알게 되었다.

 > **재훈**: 변수 호이스팅 부분을 더 다시 새롭게 알게 된 것 같았고,
