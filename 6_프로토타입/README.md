## 정리

```
혜성: 프로토 앞뒤로 붙은 거 던더프로토가 뭔지 몰랐는데 이번 책을 읽으면 상위 프로토타입을 참조하는 프로퍼티임을 알게 됨
포토타입 체인을 따라서 최상단에 Object.prototype인 것이 중요함
프로포토타입이나 던더프로토 생략 가능 아이디어가 재밌음
this 바인딩과 관련해서 알게 됨
Object.creat가 중요함
```

```
재훈: 프로토타입 공부하면서 어려움
새롭게 알게된 점, 흔히 써오던 메서드같은 것도 던더프로토가 생략되어있음 알게됨
중간에 예제 속 콘솔창에 출력 결과에서 짙은 색 옅은 색 변수들 무슨 차이인지 몰랐는데 책에서 enumerable, innumerable의 차이임을 알음
constructor 프로퍼티 부분에서 어떤 인스턴스 생성자 정보를 알아낸 것에 항상 안전한 것이 아님이 새로움
프로토타입 체인 부분에서 메서드 오버라이드 부분에서 새로움이 있었음
```

```
은지: 그림 하나 이해하는 것이 중요 165쪽 6-5
프로토타입 체인 부분에 코드가 너무 어려움
프로토타입 개념 정도만 이해함
```

```
진우: 프로토타입 개념이 중요하다라고 생각
프로토타입의 개념 한 문장이 중요
생성자 인스턴스 프로퍼티 용어 앞에 3장 this 생성자 내부함수 이해가 잘 안 됨 왜 생성자 함수를 만들었는데 this가 왜 있지? 3장 개념 부족을 느낌
말이 어려워서 그림 보고 이해하는 데 오래 걸림 / 코드 보면서 이해에 있어 도움이 됨
프로토타입안에 있는 메서드를 쓸 수 있다는게 흥미로움
```

```
찬균: 프로토타입의 개념이해 그림이 반가웠는데 그림을 이해하지 못 함
프로토타입 메서드로 던더프로토가 접근할 수 있음 혼돈
메서드 오브 개념은 알지만, 언제 사용하는지 왜 필요한지 혼란 스러움
프로토타입 체인부터는 이해하는 것에 어려움
```

```
유진: 다양한 constructor 접근법 중요
메서드 오버라이딩 개념 자바에서 배웠었는데 반가움
프로토타입 생성자 인스턴스가 낯설어서 잘 안 읽힘
프로토타입 도식 그림이 어려웠으나 나중에는 이해가 됨
```

```

성은: 프로토타입 개념에 대해서 이해
던더프로토를 생략하면 this 가리키는 것 헷갈리고 중요
Object.create(null) 새로움
무한 체인 흥미로움
```

```

효중: 배열 선언 push, pop 메서드의 호출이 던더프로토가 생략과 Array.prototype의 참조가 연관 있다는 것을 깨달음
constructor 생성자 함수를 찾을 수 있음이 새로움
프로토타입 체인 끝이 Object.prototype라는 사실을 새롭게 알게됨
다중 프로토타입은 사실 잘 안 읽힘
```

```

세희: this를 인스턴스로 할 수 있는 방법으로 던더프로토 없이 인스턴스에서 곧바로 메서드를 쓰는 것이 중요함
constructor 프로퍼티, 원래의 생성자 함수(자기 자신)를 참조하는 프로퍼티로 인스턴스로부터 그 원형이 무엇인지를 알 수 있는 수단이 중요함
프로토타입 체인 어떤 데이터의 던더프로토 프로퍼티 내부에 다시 던더프로토 프로퍼티가 연쇄적으로 이어진 것으로 헷갈림
```

```js
var Person = function (name) {
  this.name = name;
};
Person.prototype.getName = function () {
  return this.name;
};
var iu = new Person("지금");
iu.getName = function () {
  return "바로" + this.name;
};
iu.getName(); // 바로 지금
iu.__proto__.getName(); // undefined
// 바인딩된 this __proto__에는 name 프로퍼티가 없기 때문
Person.prototype.name = "이지금";
iu.__proto__.getName(); // 이지금
iu.__proto__.getName.call(iu); // 지금
```
